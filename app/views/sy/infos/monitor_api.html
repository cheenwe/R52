系统异常监控 api

根据程序定时推送上来的状态, 来判断程序的运行状态是否异常.

主要功能:

  1. 接收程序推送上来的 api 数据

  2. 自动判别程序是否异常

    • a. 实时推送时判断异常
    • b. 定时检查超过 [5 分钟] 位推送判断异常

  3. 自动邮件通知功能

  4. 所有参数支持配置

表单设计

rails g scaffold sy/app name:string abbr:string

rails g scaffold sy/service app_id:integer name:string state:integer abbr:string remark:string is_open:boolean

rails g scaffold sy/alarm service_id:integer alarm_at:datetime reason:string

rails g scaffold sy/config var:string value:string

推送接口

HTTP请求方式&URL

  • POST api/v1/heartbeat

请求参数

参数 必选 类型 字段名 说明
token ture string 传入token 目前仅校验 8 位以上随机字符串
t ture string 当前时间戳 精确到毫秒 13 位, 如:
app ture string 应用名 标识符 如: 报文 bw
service ture integer 服务 标识符 如: 报文 shibie
state ture integer 程序状态 0: 正常 1: 异常
multi false integer 是否推送多个服务 可选, 如果传输多个需要传入 1
reason false string 状态异常原因 可选
services false arrary 推送多个服务状态 见示例

返回字段

返回字段 字段类型 说明
code http返回结果状态。200：成功；409: 参数错误。
success 是否成功, 1 成功 0 失败。
msg 系统返回成功或失败说明
